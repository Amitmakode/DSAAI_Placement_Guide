# 📋 Introduction SOP — Placement Readiness Program

## Standard Operating Procedure

**Introduction — Structure, Points to Exclude & Do's / Don'ts**

---

## Section A — Introduction Structure (Follow this Order)

| # | Section / Component | Guidance Notes |
|---|---------------------|----------------|
| 1 | **Name** | State your full name clearly. *(Optional — skip if already introduced by the interviewer)* |
| 2 | **Education** *(Reverse Chronological)* | Mention degrees in reverse order — most recent qualification first. |
| 3 | **Experience** *(Reverse Chronological)* | Describe work history in reverse order — most recent role first. |
| 4 | **Roles & Responsibilities** | This is the **CORE** of your introduction. Describe what you actually did — datasets analysed, models built, pipelines designed, tools used (Python, SQL, ML frameworks), insights generated, etc. |
| 5 | **Key Skills** | List relevant technical skills & tools: Python, SQL, Tableau, Power BI, ML libraries, cloud platforms. Keep it concise — do not explain each skill. |

---

## Section B — Points to Exclude from Introduction

| ❌ Item to Exclude | Why It Must Be Removed |
|--------------------|------------------------|
| ✗ Hobbies | Personal interests are irrelevant in a professional Data Science introduction. |
| ✗ Strengths & Weaknesses | This is for HR rounds — never volunteer it in an intro. |
| ✗ Family Details | Personal / family background has no place in a job introduction. |
| ✗ Long-Term & Short-Term Goals | Goals are discussed only when specifically asked by the interviewer. |
| ✗ Extra-Curricular Activities | Keep the introduction strictly professional and role-focused. |

---

## Section C — Do's & Don'ts for Introduction

| ✅ DO'S | ❌ DON'TS |
|---------|-----------|
| ✓ Use past tense or present continuous tense *(e.g., "Built ML models" / "Building data pipelines")* | ✗ Don't use phrases like "I can...", "I will...", "I want to..." |
| ✓ Keep introduction duration to a **minimum of 5 minutes** | ✗ Don't explain what a technology is — state what you did with it |
| ✓ Focus primarily on **Roles & Responsibilities** | ✗ Don't use client names or project names |
| ✓ Speak with clarity, confidence and good pace | ✗ Don't divide responsibilities by tool *(e.g., "For Python I did… For SQL I did…")* |

---

## ⚡ Key Reminder

> **Minimum 5 minutes introduction** · Always focus on **WHAT YOU DID**, not what the technology is · **Past tense or present continuous tense only** · **No client / project names**
